Signed vs Unsigned Observations: Building Trust in Data

June 28, 2025 • 7 min read • Technical

In a world of deepfakes and misinformation, how do we know what's real? The CIN's approach to data verification distinguishes between signed observations (with verified sources) and unsigned observations (anonymous or unverified).

The Trust Crisis in Digital Information

As AI-generated content becomes increasingly sophisticated, distinguishing between authentic and synthetic information grows more challenging. This creates a fundamental trust problem: How can we build reliable systems when we can't trust the data they're built on?

The CIN addresses this challenge through a dual-layer approach to data: signed observations and unsigned observations.

Key Insight

Rather than treating all data as equally trustworthy, the CIN explicitly tags data with its provenance and verification status, allowing systems to make appropriate trust decisions based on the context and requirements.

Signed Observations: Verified and Accountable

A signed observation in the CIN has these key characteristics:

  • Cryptographic Signature: The observation is cryptographically signed by the observer using their private key, creating a tamper-evident record.
  • Identity Binding: The signature is linked to a Decentralized Identifier (DID) that represents a known entity (person, organization, or device).
  • Accountability: The observer is accountable for the accuracy of their observation and can face reputation consequences for false claims.
  • Verifiability: Anyone can verify who made the observation and when it was made.

Example: Signed Observation Structure

{
  "id": "obs_1a2b3c4d5e",
  "observer": "did:cin:alice",
  "timestamp": "2025-06-15T14:32:18Z",
  "observation": {
    "type": "temperature",
    "value": 22.5,
    "unit": "celsius",
    "location": {
      "lat": 37.7749,
      "lng": -122.4194
    }
  },
  "signature": {
    "type": "Ed25519Signature2020",
    "verificationMethod": "did:cin:alice#key-1",
    "signatureValue": "z58Da3kp...Lk4Mw=="
  }
}

Unsigned Observations: Anonymous but Useful

Unsigned observations lack cryptographic signatures and identity binding but still have value:

  • Anonymity: The observer's identity is not disclosed, protecting privacy.
  • Lower Trust Weight: Systems assign lower confidence to unsigned observations.
  • Aggregation Value: While individual unsigned observations may be unreliable, patterns across many unsigned observations can reveal useful insights.
  • Whistleblower Protection: Allows reporting of sensitive information without fear of retaliation.

Example: Unsigned Observation Structure

{
  "id": "obs_9z8y7x6w5v",
  "timestamp": "2025-06-15T14:35:42Z",
  "observation": {
    "type": "temperature",
    "value": 22.7,
    "unit": "celsius",
    "location": {
      "lat": 37.7749,
      "lng": -122.4194
    }
  },
  "confidence": {
    "source": "anonymous",
    "level": "unverified"
  }
}

Practical Applications

This dual approach to observations enables several powerful capabilities:

1. Contextual Trust Decisions

Systems can apply different trust thresholds based on the context:

  • For critical medical data, only signed observations from verified healthcare providers might be accepted.
  • For crowd-sourced traffic information, a weighted blend of signed and unsigned observations might be used.
  • For sensitive whistleblowing, unsigned observations might be accepted but flagged for additional verification.

2. Reputation-Based Verification

The CIN's reputation system tracks the historical accuracy of observers, allowing:

  • Higher weights for observers with strong track records of accurate observations.
  • Automatic downgrading of observers who consistently provide inaccurate information.
  • Domain-specific reputation scores (e.g., an observer might be highly trusted for weather data but not for medical diagnoses).

3. Consensus Mechanisms

When multiple observations conflict, the system can:

  • Prioritize signed observations from high-reputation sources.
  • Look for consensus among independent observers.
  • Flag discrepancies for human review when critical decisions are involved.
  • Apply statistical methods to identify outliers and potential misinformation.

Conclusion: A Foundation for Trustworthy AI

By explicitly distinguishing between signed and unsigned observations, the CIN creates a foundation for trustworthy AI systems that can:

  • Make appropriate trust decisions based on data provenance.
  • Protect privacy while still enabling valuable anonymous contributions.
  • Create accountability for verified information sources.
  • Build resilience against misinformation and synthetic content.

This approach doesn't eliminate the challenge of digital trust, but it provides a framework for managing it in a nuanced, context-aware manner that balances the competing needs for verification, privacy, and inclusivity.
